## Output Template: Jira Ticket

Generate a Jira ticket using the following structure. Do not add sections beyond what is listed.

**Description**
One to three sentences describing the need or problem being solved. Focus on what and why, not how.

**Acceptance Criteria**
A numbered list of goal-oriented criteria. Each item describes an observable outcome when the ticket is complete. Do not include implementation steps, technical instructions, or "how to" language. Do not include any coding sampeles

**Technical Notes**
Bulleted list of relevant technical context, constraints, or background from the meeting. Do not attribute notes to specific individuals. Present as factual observations. Include references to software mentioned in the context, for example if a specific node library or application is references.
