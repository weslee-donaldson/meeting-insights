# planning/ — Design documents and requirements

Long-lived planning documents that capture the functional requirements and visual design system for the application. These are reference material, not runtime config.

## Files

| File | Purpose |
|------|---------|
| `base-functional-requirements.md` | Core feature specifications and user stories — defines what the application does, organized by domain area (ingestion, search, threads, insights, timelines, action items) |
| `design-system.md` | Visual design guidelines, component specifications, and spacing/typography rules extracted from Paper artboards; referenced by `design-tokens.ts` |
