# UI Source

## Design System

All UI code in this directory follows a design system defined in Paper artboards (`Meeting Insights.paper`).

### Key Files
- `design-tokens.ts` — Single source of truth for spacing, typography, radii, component specs. Import from here, never hardcode.
- `index.css` — Theme CSS variables (`[data-theme="stone-light"]`, etc.). Components use `var(--color-*)` references.
- `theme.ts` / `ThemeContext.tsx` — 4 themes: stone-light, stone-dark, teal-light, teal-dark.

### Paper MCP Verification (mandatory for UI bursts)
1. Before implementing: `get_screenshot` the artboard referenced in the ketchup-plan section
2. After implementing: Playwright MCP screenshot at 2560×1440, compare against artboard
3. Fix discrepancies before committing

### Accessibility
3-tier text system (see `design-tokens.ts` → `textTiers`):
- Primary/Body: 7:1+ (`--color-text-primary`, `--color-text-body`)
- Secondary: 4.5:1+ (`--color-text-secondary`)
- Muted: 3:1+, labels at 11px+ bold only (`--color-text-muted`)
- Decorative: borders/dividers only, NEVER for text (`--color-line`)

### Layout Pattern
Every page follows: **nav rail (56px) | sidebar list | detail panel | assistant chat (380px)**. No exceptions. See "Concept B — Action Items" artboard.
